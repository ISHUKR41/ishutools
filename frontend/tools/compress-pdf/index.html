Everything About PDF Compression

What is PDF Compression?

PDF compression reduces the file size of a PDF document by reprocessing its internal data streams. A PDF file consists of objects — image streams, font streams, content streams, and metadata. Compression works by reencoding these streams with better algorithms, removing redundant data, deduplicating identical objects, and optionally reducing image resolution.

IshuTools by Ishu Kumar (ISHUKR41) uses a 10-engine pipeline to guarantee the smallest possible result. Unlike single-engine tools, we try all strategies and pick the winner.

The Lossless preset at ishutools.fun is unique — it is the only mode that guarantees zero image re-encoding. Ghostscript and PyMuPDF are strictly disabled for lossless/high presets. Images are bit-for-bit identical in the output.

Technical Details — How It Works

  • Image streams — JPEG re-encoding at controlled quality + DPI downsampling (screen/low/medium only)
  • Stream recompression — DEFLATE-9 (zlib level 9) on all non-image streams
  • Object stream merging — Multiple objects packed into ObjStm containers
  • XRef stream — Compact cross-reference table vs. traditional section
  • Font subsetting — Only embed used glyphs via Ghostscript
  • Deduplication — MD5 hash-based identical image removal
  • Orphan removal — Dead objects not referenced by page tree
  • Metadata strip — Remove XMP + DocInfo (optional)
  • Linearization — Fast-web-view byte-range serving (optional)
