Frequently Asked Questions

How much can I compress a PDF online free with IshuTools?
IshuTools can reduce PDF size by up to 90% with the Screen preset (72 DPI). Image-heavy PDFs compress the most. Text-only PDFs see 5–25% with Lossless. Try different presets — the tool shows estimated reduction for your specific file after analysis.
Does compressing PDF reduce quality?
Only if you choose Screen or Low presets which downsample images. The Lossless preset makes zero image quality changes — it only compresses PDF structure, streams and metadata. The High preset (200 DPI) also maintains excellent visual quality. IshuTools never automatically converts to grayscale — that is only done if you explicitly enable it in Advanced Options.
How to compress PDF to 200KB or less?
Enable Target Size Mode in Advanced Options and enter 200 in the KB field. IshuTools uses binary-search compression, trying progressively aggressive presets until the output is within 5% of your target. Note: very aggressive size targets may require significant quality reduction depending on the source PDF.

Is my PDF data safe on IshuTools?
Yes. Files are processed in isolated temporary server directories and are never stored permanently. No third parties access your document. Files are automatically deleted after your session. HTTPS encrypts all transfers. IshuTools is a privacy-first tool by Ishu Kumar.
Which compression level should I choose?
Screen: Maximum compression (72 DPI). Great for web-only viewing where file size matters most.
Low: 96 DPI — good for email attachments.
Medium (Recommended): 150 DPI — best balance. Works for most use cases.
High: 200 DPI — excellent quality, meaningful compression. Good for professional documents.
Lossless: Zero image change — only PDF structure optimized. Ideal for archival and print documents.

How IshuTools PDF Compression Works — Technical Details

Ghostscript (/screen, /ebook, /printer, /prepress): Industry-standard PostScript interpreter. Re-renders and re-encodes the PDF with BICUBIC downsampling to target DPI. /screen produces smallest file (72 DPI), /prepress produces print-quality output.

PyMuPDF (fitz): Extracts each embedded image individually, applies LANCZOS resampling to target DPI, progressive JPEG re-encoding. Only saves if the new version is actually smaller.

pikepdf: DEFLATE-9 object stream compression, metadata stripping, annotation removal, XFA form removal, JavaScript stripping, thumbnail deletion, ICC profile removal — all user-controlled, never automatic.

qpdf: Stream recompression at maximum DEFLATE level, optional linearization for fast web viewing.
